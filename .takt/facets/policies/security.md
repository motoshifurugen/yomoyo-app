# Policy: Security

`.claude/rules/security.md` を要約・強制するポリシー。セキュリティはブロッキング条件。

## コミット前チェック（いずれか未達ならコミット不可）
- [ ] ハードコードされた秘密情報が無い（APIキー・パスワード・トークン）
- [ ] 全外部入力が検証されている
- [ ] SQL インジェクション対策（パラメータ化 / ORM）
- [ ] XSS 対策（サニタイズ、危険なレンダリング無し）
- [ ] CSRF 対策（SameSite クッキー / CSRF トークン）
- [ ] 認証が正しく実装されている
- [ ] 保護リソースごとに認可が強制されている
- [ ] 公開エンドポイントにレート制限
- [ ] エラーメッセージが機微情報を漏らさない

## 秘密情報管理
- 秘密情報はハードコードせず環境変数から読む。`.env` はコミットしない。
- リポジトリは将来公開され得る前提。必須の秘密情報が無ければ明示的に失敗させる。

## 認証・認可
- 認証と認可は別概念。保護リソースは必ず認可を強制。クライアント側のロール判定に依存しない。
- 不明なときはデフォルト拒否。

## エラーハンドリング
- ユーザー向けエラーは汎用で安全に。スタックトレース・クエリ・内部IDを露出しない。
- 詳細は内部ログのみ。

## インシデント対応
セキュリティ問題発見時は機能作業を停止し、CRITICAL を先に修正、露出した秘密情報を即ローテート、
同種問題を全体点検し、再発防止策（lint/フック/チェック）を追加する。
