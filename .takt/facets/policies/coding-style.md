# Policy: Coding Style

`.claude/rules/coding-style.md` を要約・強制するポリシー。

## 不変性（CRITICAL）
関数引数・共有状態・React state/props・公開APIでは不変パターンを優先する。
入力や共有オブジェクトをミューテートしない。関数境界を越えないローカルな
ミューテーションのみ許容。

## ファイル構成
- 多数の小さく焦点の絞れたファイルを優先（高凝集・低結合）。
- 200–400行が理想、800行は警告閾値。
- 大きなコンポーネントからユーティリティを抽出する。
- 技術種別ではなく機能/ドメインで構成する。

## エラーハンドリング
- 適切な境界で捕捉する。承認されたロギング機構を使う。
- 意味のある、ユーザー向けのメッセージを再スローまたは伝播する。

## 入力バリデーション
- API リクエスト・ユーザー入力・外部サービスのデータを検証する。
- スキーマベース（例: zod）を優先。内部の純粋関数での冗長な検証は避ける。

## 完了前チェックリスト
- [ ] 可読で良い命名 / 関数 <50行 / ファイル <800行 / ネスト深さ <=4
- [ ] 適切なエラーハンドリング / デバッグログ無し
- [ ] マジックナンバー・ハードコード設定無し / 共有データに不変パターン
