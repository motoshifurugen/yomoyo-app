# Policy: Testing

`.claude/rules/testing.md` を要約・強制するポリシー。

## カバレッジ
- 目標 80%。計測ツールがある箇所では必須。ツール不在は進行を妨げないが文書化する。
- カバレッジ数値は意味あるテストケースの代替にはならない。

## 必要なテスト種別（変更影響に応じて）
1. Unit: 関数・ユーティリティ・フック・コンポーネント（ロジック変更では必須）
2. Integration: API・DB・サービス境界（データフロー/副作用が絡む場合に必須）
3. E2E: 重要ユーザーフローのみ（認証・決済・コア体験で必須）

## TDD（デフォルトワークフロー）
RED（失敗テスト）→ 失敗確認 → GREEN（最小実装）→ 通過確認 → IMPROVE（リファクタ）→ 再実行。
新機能・ビジネスロジック・バグ修正では TDD 必須。

## 例外（PR に明記すれば省略可）
- 振る舞いを変えない純粋なリファクタ
- コメント / ドキュメントのみ
- 型のみ / 設定のみの変更

## 失敗時の対応
テスト分離を確認 → モック/フィクスチャを点検 → 実装を先に直す →
テストが誤りと証明された場合のみテストを変更する。
